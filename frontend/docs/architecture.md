# System Architecture & Multi-Agent Pipelines

The architectural backbone of **DevCareer Intelligence** is meticulously designed to isolate asynchronous long-horizon AI processes explicitly away from blocking the synchronous client-side experience. Analyzing vast multi-file enterprise code repositories, filtering for tutorial-clone similarities, rewriting large unstructured PDF paragraphs, and performing live market analysis requires dozens of chained API inferences across Groq, Gemini, and GitHub APIs. 

If this heavy workload operated in a generic HTTP request bound context, edge proxies would ungracefully invoke absolute 504 Gateway Timeout errors repeatedly. The solution relies structurally down to the core utilizing state-management queues, resilient databases, and Next.js boundary handlers to seamlessly create a highly fault-tolerant pipeline.

---

## 1. Topological Overview

At its core, the application breaks down logically into three specific topological subsystems:

### A. The Client Facing Application (Next.js)
Residing within standard modern `React Server Components (RSC)`, it manages file parsing UI, uploading context, streaming the actual Terminal-level server progress updates down utilizing standard network hooks, and compiling the immense JSON trees generated by the AI into highly visual robust mathematical graphics (Skill Dimensions Radar, Timeline components, Markdown syntax highlighting).

### B. Persistent Database Integration (Supabase PostgreSQL)
Supabase acts heavily as the state bridging machine crossing information between the stateless UI endpoints scaling up against the stateful background worker instances processing the tasks linearly holding persistence layers.

### C. Background Orchestrator System (BullMQ mounted in Node)
The absolute workhorse. Driven locally by an embedded native Node array loop and Redis maps (in scalable deployments), the background network implements `BullMQ` natively across separated processing clusters named `github-fetcher`, `code-analyzer`, `ai-synthesizer`, and `market-fetcher`.

## 2. Database & State Modeling Structure

To securely and effectively manage analytic depth without polluting memory graphs, everything heavily relies completely around structured `Postgres` indexing paths defined comprehensively across Supabase relations:

### Core Tables List
1. **`audit_sessions`**: The absolute transactional node context. Manages the overall processing boundaries (`progress_percent`, `status`) and tracks the unique generated `session_id`. When the NextJS client requests an audit, it instantly binds visually backward matching this row, constantly parsing state.
2. **`repo_analyses`**: The detailed breakdown of repository analytic vectors. Specifically logs `api_design_score`, `complexity_tier`, `security_issues` JSON boundaries, and `plagiarism_flags`. Designed dynamically holding granular multi-repo analysis sequentially mapped around the single audit tracking array.
3. **`skill_profiles`**: Analyzed gap maps calculating standard benchmarks, penalizing strictly around heavily copied boundaries. Evaluates boundaries crossing specifically into `DB Design`, `Security`, and `Frontend` calculations to dynamically project `Verified Overall Tiers`.
4. **`roadmaps`**: Saves strictly formulated AI synthesized arrays bridging 13 sequential weekly boundaries containing deep prescriptive task schemas pointing toward the `rewritten_bullets` array designed for the native UI Resume Renderer.
5. **`market_fit`**: Links directly verified tiers connecting user parameters against active real-world hiring environments returning arrays categorized strictly through `qualify_now`, `qualify_90day`, and `qualify_6mo` bins.

## 3. The 4-Stage Multi-Agent BullMQ Pipeline

Instead of a single immense blocking functional layer executing thousands of instructions logically synchronously, jobs traverse highly separated queued pipelines triggering precisely sequential states explicitly bounded by specialized functional logic engines handling retries securely internally safely. 

### Stage 1: The Codebase Retrieval Engine (`jobs/github-fetcher.ts`)
When the primary `/api/audit/start` controller is fired, the database allocates the `target_urls` scaling against `target_branches` to load the exact environment and dispatches an intensive job into `fetch-repos`.
- **Strategic Downloading**: Avoids massively cloning `node_modules`, standard `/dist`, output binaries, and massive `SVG/Images`. It targets specifically strictly typed application layer syntaxes.
- **Commit History Tracking**: Triggers direct `getCommitsByAuthor` heuristics explicitly isolating specific developer pushes inside collaborative environments mapping strictly only mathematical boundaries written completely uniquely to the applicant.
- **Next Chain Dispatchable Event**: Triggers queue entry pushing `analyze-repo`.

### Stage 2: The Core Inference Analyzer (`jobs/code-analyzer.ts`)
Once data trees correctly resolve inside local ephemeral environments physically allocating the raw blocks, the engine conducts intensive mathematical structural abstractions querying directly through Groq logic interfaces predicting patterns.
- **The Plagiarism Firewall (Tutorial Clones)**: Dynamically checks explicit file logic boundaries querying basic "Calculator.zip", "TicTacToe" or typical "React Todo App" clone traces mapping their presence implicitly. Analyzes strict commits across multiple days versus singular copy-paste dumps defining absolute `confidence` probabilities applying aggressive structural negative weighting parameters if flagged successfully.
- **Architectural Bounding**: Splits immense code architectures utilizing sliding sliding character window tokens protecting overarching API endpoint token limitations scaling chunks efficiently cross-referencing input validation logic seamlessly parsing API patterns and abstract modular separations producing specific syntax findings bounding the JSON schemas explicitly.
- **Next Chain Dispatchable Event**: Modifies session rows globally tracking up triggering `synthesize-profile`.

### Stage 3: The Resume Generator Synthesizer (`jobs/ai-synthesizer.ts`)
Responsible heavily for overarching structural gap analytics binding the explicit resume formatting outputs.
- **Raw Extract Buffer Re-Alignment Engine**: Evaluates standard resume PDFs explicitly generating raw linear textual output streams which are heavily prone to `pdf-parse` failing byte alignments missing standard Unicode list operators (creating strings combining specific text segments). By utilizing deeply explicit mathematical matching loops `(?:\n|•|\*|—|-(?=\s)|210\s+)`, it explicitly targets standard numeric structural `210` bullets effectively extracting specific paragraphs efficiently mapping user claim definitions.
- **Truth Generator Rewriter Prompt**: Utilizing Llama 4 frameworks pushing the original unverified user claims across strictly tested actual evidence variables produced earlier returning highly concise evidence-based rewritten lists scaling alongside matching structural explicit metrics.
- **Prescriptive 90-Day Output Module AI**: Projects a highly localized JSON schema delivering an intense targeted prescriptive curriculum addressing explicitly identified `Security` or `Database Design` knowledge voids strictly defined within algorithmic bounds matching 13 weekly phases correctly.
- **Next Chain Dispatchable Event**: Safely executes parallel `upsert` inserts utilizing standard primary schemas limiting rows and triggering globally active `fetch-market`.

### Stage 4: Regional Live Market Query Aligner (`jobs/market-fetcher.ts`)
A completely isolated data acquisition module avoiding massive computation algorithms heavily focused specifically upon aggregating dynamic structural arrays accessing exact location boundaries matching actual candidate requirements seamlessly connecting directly to live Job Board Restful Integrations.
- **Data Cross-Referencing**: Translates the parsed "Verified Skill Map" specifically extracting boundaries querying generic languages recognized physically inside the github repository parameters crossing dynamically scaling API boundaries matching native Jooble schemas and Remotive specific logic loops seamlessly handling fallback logic seamlessly.
- **Completion Cycle Hook**: Executes the `session` array correctly identifying the overall state structure modifying dynamically the native `status` effectively signaling directly down toward the Front-End Client polling architectures pushing users automatically scaling seamlessly across specific URL paths launching highly scalable `ReportDashboardClient` UI render views accurately rendering JSON parameters beautifully dynamically executing structural component layouts natively displaying complex intelligence profiles correctly bridging candidate testing capabilities.

## 4. Resilience & Scaling Philosophies

- **Failure Fallbacks:** Any failed model query (Groq rate-limit errors triggering `429 Too Many Requests`) elegantly falls directly into `Google Gemini` scaling structural capabilities handling multi-token array requests efficiently blocking failure states.
- **Database Transaction Constraints:** Operations performing native modifications limit internal Supabase requests explicitly binding native `crypto.randomUUID()` paths enforcing isolated row limits handling single transactions avoiding "PGRST116 multiple rows returned" failure boundaries preserving query resilience continuously validating architectures sequentially natively executing operations optimally achieving perfect isolation environments maintaining total transaction tracking integrity.
